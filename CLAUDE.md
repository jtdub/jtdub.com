# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Commands

```sh
pnpm install              # install dependencies (Node 22+, pnpm 10+)
pnpm dev                  # dev server at localhost:4321
pnpm build                # build to dist/, run pagefind indexing, copy index to public/
pnpm preview              # serve the built dist/ directory
```

Search only works after at least one `pnpm build` (Pagefind indexes post-build HTML, then copies the index to `public/pagefind/` for the dev server).

CI (.github/workflows/ci.yml) runs `pnpm build` and verifies `dist/` exists. No test suite.

## Architecture

This is a personal site built with Astro 6, statically generated and hosted on Cloudflare Pages.

### Dual-zone design system

The site has two editorial zones — **Field & Research** (anthropology, archaeology, fieldwork) and **Engineering & Code** (network automation, OSS, Python). They share typography and layout but swap accent colors via a `data-zone` attribute on `<html>`:

- `tokens.css` defines default colors plus `[data-zone='field']` (terracotta) and `[data-zone='engineering']` (ink-blue) overrides on `--accent`, `--accent-deep`, `--accent-tint`
- `BaseLayout.astro` sets `data-zone` from the `zone` prop; zone `'home'` omits the attribute (uses default ink accent)
- `src/utils/zone.ts` provides `zoneLabel(zone, 'short' | 'long')` for display text

All accent-dependent styling uses `var(--accent)` — never hardcode zone colors.

### Content collections

One collection: `posts` (135 markdown files in `src/content/posts/`). Schema in `src/content.config.ts`:
- Required: `title`, `date` (coerced to Date)
- Optional: `author`, `tags` (string[]), `description`, `zone` (enum, defaults 'field'), `external_url`

Post filenames must be `YYYY-MM-DD-slug.md`. The regex in `src/utils/post-url.ts` (`parsePostId`) enforces this — breaking it will fail the build.

### Post URL pattern

Posts render at `/:year/:month/:day/:slug/` via `src/pages/[year]/[month]/[day]/[slug].astro`. This matches the old Jekyll permalink pattern for backward compatibility — do not change it without a redirect strategy.

### Data-driven pages

`src/data/*.json` files (resume, publications, academic papers, projects, photos) are imported directly in pages — no content collection layer. Pages transform the data on-page (e.g., photography groups photos by year, resume renders sections from nested arrays).

### Fonts

Self-hosted via `@fontsource-variable` (Fraunces, Inter, JetBrains Mono). Imported in `BaseLayout.astro`. Font-family names in `tokens.css` use the `Variable` suffix (e.g., `'Fraunces Variable'`).

### Search

Pagefind-based. `SearchModal.astro` lazy-loads `/pagefind/pagefind-ui.js` and CSS on first open (triggered by clicking search button, pressing `/`, or `Cmd+K`). Uses `<script is:inline>` to avoid Vite bundling. The standalone `/search` page also exists as a fallback.

### Images

All photos are served via Cloudflare Images (`imagedelivery.net`). Photography grid uses `/w=640,fit=cover` variant for thumbnails; lightbox loads the `/public` (full-resolution) variant.

## Key conventions

- **CSS**: BEM-ish naming with page-specific prefixes (`.article__*`, `.zone-landing__*`, `.post-list__*`). Each `.astro` file uses scoped `<style>` blocks. Shared styles live in `src/styles/` (tokens, base, article).
- **Scripts**: Use `<script is:inline>` for vanilla JS that must avoid Vite bundling (SearchModal, lightbox, Chart.js pages). Use `var` in inline scripts (not processed by the TS compiler).
- **Utilities**: `src/utils/post-url.ts` (parsePostId, postUrl), `src/utils/tag-slug.ts` (tagSlug), `src/utils/zone.ts` (zoneLabel). Use these instead of inlining the same logic.
- **Tag slugs**: Generated by `tagSlug()` — lowercase, spaces to dashes, strip non-alphanumeric. Both `tags/index.astro` and `tags/[tag].astro` must use the same function or tag links will 404.
- **RSS**: `src/pages/feed.xml.ts` at `/feed.xml`. Must stay at this path (subscribers depend on it).
- **iOS app URLs**: `/apps/privacy/{slug}` and `/apps/support/{slug}` are required by Apple for App Store listings. Do not remove or rename without updating the App Store entries.
